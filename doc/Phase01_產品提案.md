# 1. 價值主張 (Value Proposition)

**一句話說故事 (Elevator Pitch):**
為按時間收費的小型顧問公司打造的輕量級專案營利追蹤工具。透過無縫整合「專案預算(時間)」與「實際工時」，讓專案經理掌控專案進度，同時顧問能快速簡易完成工時回報。
**價值:** 消除專案中的「隱形工時」與「過度服務 (Over-servicing)」，讓時間真正與金錢掛鉤。

# 2. 目標客群與痛點分析 (Target Audience & Pain Points)

本專案針對小型顧問公司（約 5-15 人規模）的兩種核心角色進行痛點解決：

### 角色 A：專案經理 / 管理層 (Admin)
* **痛點：** 現有專案管理工具多專注於「任務狀態 (To-Do/Doing/Done)」，缺乏時間維度的結合，然而，時間對顧問公司來說就是金錢。
* **解決：** 需要在專案預算（時數）燃燒殆盡前收到警示，以確保專案不虧錢，並能清晰量化員工產出的 KPI。

### 角色 B：顧問 / 執行人員 (Employee)
* **痛點：** 填寫工作日誌是繁瑣的額外負擔，且往往不知道自己在該專案還剩下多少時間配額可以使用。
* **解決：** 需要一個能清楚知道「我還剩多少時間可以做這件事」的參考指標，並能快速完成工時回報。

# 3. MVP 解決方案與核心功能 (Core Features)

為了快速驗證市場價值，MVP (最小可行性產品) 將專注於以下核心功能：

1.  **任務時數配置：** 根據報價單下拆解任務，賦予預估時數。
2.  **工時回報介面：** 顧問針對特定任務輸入實際耗用工時。
3.  **進度儀錶板：** 即時動態比對「總預算時數」與「已用時數」，透過視覺化圖表讓專案經理掌握專案，顧問看見剩餘時數配額。

前置作業
4.  **客戶與專案:** 盤點客戶資源，開啟潛在專案。
5.  **報價單:** 報價單的細項，包含各任務與金額。

# 4. 商業邏輯與業務流轉 (Business Logic & Work Flow)

此流程將作為 Phase02 的基礎：

1.  **[起始] 標準工作：** 客戶 -> 專案: 開會釐清需求 -> 報價單: 細項包含任務與金額。
2.  **[規劃] 任務預算：** 將報價單 -> 轉化為有預算時數的任務。(金額 / 顧問每小時收費 = 時數)。
3.  **[執行] 工時消耗：** 顧問執行任務 -> 填寫工時表。
4.  **[監控] 狀態同步：** **顧問端：** 即時查看該任務/專案的「剩餘可用時數」;**專案經理端：** 透過儀表板檢視「專案進展」與「時間進展」

# 5. 成功指標 (Success Metrics / KPIs)

為衡量此 MVP 是否達成商業目標，定義以下追蹤指標：
    * **顧問端：** 工時表按時填報完成率 (Timesheet Completion Rate) > 85%。
    * **管理端：** 專案預算超支率 (Budget Overrun Rate) 下降幅度。

# 6. 未來藍圖 (Future Roadmap)

在 MVP 驗證成功後，預計導入以下功能深化產品價值 (階段性開發)：

* **Phase-Next A (快速簡易工時回報)：** 與現有專案管理清單 (如 Jira, Asana, Trello) 深度整合，自動帶入任務資訊，減少填寫工作日誌的重複工作量。
* **Phase-Next B (自動化與體驗升級)：** 導入 Nudge (輕推) 機制。透過彈出訊息/Slack Bot 提醒休息的同時簡易工時回報。
* **Phase-Next C (財務一體化)：** 未來加入請款單模塊，從「報價單 -> 工時追蹤 -> 分期與請款單」的自動化。


## 7. 競品借鑑 (Competitor Reference: Teamleader)

* **借鑑點：** 荷蘭的 Teamleader 完美實現了從 CRM 報價到專案管理，再到 Invoice 請款的「一條龍資料不斷鏈」。
* **我們的差異化切入 (MVP 策略)：** Teamleader 體積龐大且設定繁瑣。我們的 MVP 將捨棄複雜的 CRM 模組，專注打透「時間/預算整合」的輕量級體驗，特別針對 30 人以下的顧問團隊，提供最快能看出「專案賺不賺錢」的捷徑。
